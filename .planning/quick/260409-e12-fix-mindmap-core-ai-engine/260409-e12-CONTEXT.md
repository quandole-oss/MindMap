## Task Boundary

Fix the four critical issues in MindMap's AI pipeline: concept routing, domain categorization, extraction quality, and misconception detection. All changes are in `packages/llm`, `packages/router`, and `apps/web/app/api/ask/route.ts`.

<decisions>
## Implementation Decisions

### Misconception Routing
- Route ALL concepts for misconception detection, not just the first
- When multiple concepts match misconceptions, diagnose the highest-confidence match
- Create one diagnostic session for the best match (not multiple sessions)

### Domain Expansion
- Expand from 5 to 15 domains: physics, biology, chemistry, math, computer-science, earth-science, astronomy, history, literature, social-studies, art, music, health, engineering, general
- Update the Zod enum in extract.ts and any downstream consumers
- "general" remains the catch-all

### Semantic Routing Fallback
- When string matching fails to find a misconception match, use LLM semantic fallback
- Ask Claude: "Does this concept relate to any known misconception?" with the misconception list as context
- Accepts ~1s additional latency per concept for better detection
- Falls back gracefully (returns enrich mode) if LLM call fails

### Extraction Quality
- Add examples to the extraction prompt showing desired granularity
- Add explicit guidance: prefer 2-4 focused concepts over 5-6 vague ones
- Include domain guidance with the 15-domain list in the prompt

### Claude's Discretion
- Specific extraction prompt wording and examples
- Similarity thresholds for semantic fallback
- How to handle edge cases (cross-domain concepts, ambiguous domains)

</decisions>

<specifics>
## Specific Ideas

- The extraction prompt should include 2-3 few-shot examples showing good vs bad concept extraction
- The semantic fallback should batch all unmatched concepts in one LLM call (not N separate calls) to minimize latency
- Router should return all matches with confidence, then the caller picks the best one

</specifics>
