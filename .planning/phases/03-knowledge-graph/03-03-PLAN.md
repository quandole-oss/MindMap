1. Create `apps/web/actions/graph.ts` ("use server"):
   - Import `auth` from "@/lib/auth", `db` and `schema` from "@mindmap/db", `eq` and `and` and `inArray` from "drizzle-orm"
   - Export interface `GraphNode { id: string; name: string; domain: string; status: "unprobed" | "healthy" | "misconception"; visitCount: number; isBridge: boolean; }`
   - Export interface `GraphEdge { source: string; target: string; edgeType: string; }`
   - Export async function `getGraphData(): Promise<{ nodes: GraphNode[]; edges: GraphEdge[] }>`:
     - Get session via `auth()`, throw if no user
     - Query all concepts for this user: `db.query.concepts.findMany({ where: eq(schema.concepts.userId, session.user.id), columns: { id: true, name: true, domain: true, status: true, visitCount: true } })`
     - Get all concept IDs from results
     - If no concepts, return `{ nodes: [], edges: [] }`
     - Query concept_edges where sourceConceptId OR targetConceptId is in the concept IDs list: use `db.select().from(schema.conceptEdges).where(or(inArray(schema.conceptEdges.sourceConceptId, conceptIds), inArray(schema.conceptEdges.targetConceptId, conceptIds)))`
     - Map nodes to GraphNode (isBridge: false for now — Plan 04 marks bridges)
     - Map edges to GraphEdge (source: sourceConceptId, target: targetConceptId, edgeType)
     - Return { nodes, edges }
   - Export async function `getNodeDetails(conceptId: string): Promise<{ concept: { name: string; domain: string; status: string; visitCount: number }; exchanges: Array<{ questionText: string; aiResponse: string | null; createdAt: Date | null }> }>`:
     - Get session via `auth()`, throw if no user
     - Query concept: `db.query.concepts.findFirst({ where: and(eq(schema.concepts.id, conceptId), eq(schema.concepts.userId, session.user.id)) })`
     - If not found, throw error (access control: only own concepts)
     - Query concept_questions + questions join: get all questions linked to this concept via conceptQuestions join table
     - Return concept info + question exchanges sorted by createdAt ASC

3. Create `apps/web/components/graph/knowledge-graph.tsx` ("use client"):
   - Props: `{ nodes: GraphNode[]; edges: GraphEdge[]; onNodeClick: (nodeId: string) => void }`
   - Use `useRef<SVGSVGElement>(null)` for SVG container
   - Use `useRef<HTMLDivElement>(null)` for tooltip container
   - State: none internally — all state managed by parent page
   - D3 constants per UI-SPEC Interaction Contract:
     - `NODE_COLORS = { healthy: "#14b8a6", misconception: "#f87171", unprobed: "#a1a1aa", bridge: "#a78bfa" }`
     - `getNodeColor(node)`: if node.isBridge return bridge, else return NODE_COLORS[node.status]
     - `getNodeRadius(node)`: `Math.min(8 + node.visitCount * 2, 24)` (per CONTEXT: capped at 24px)
   - `useEffect` (deps: [nodes, edges, onNodeClick]):
     - Guard: if !svgRef.current or nodes.length === 0, return
     - CRITICAL: Clear SVG first: `const svg = d3.select(svgRef.current); svg.selectAll("*").remove();`
     - CRITICAL: Deep-clone nodes and links before passing to D3 (per RESEARCH pitfall 7): `const simNodes = nodes.map(n => ({ ...n })); const simLinks = edges.map(e => ({ ...e }));`
     - Get container dimensions via `svgRef.current.getBoundingClientRect()`
     - Create simulation: `d3.forceSimulation<SimNode>(simNodes)`
       - `.force("link", d3.forceLink<SimNode, SimLink>(simLinks).id(d => d.id).distance(80))`
       - `.force("charge", d3.forceManyBody().strength(-120))` (per UI-SPEC: -120)
       - `.force("center", d3.forceCenter(width / 2, height / 2))`
       - `.force("collision", d3.forceCollide<SimNode>().radius(d => getNodeRadius(d) + 4))` (per UI-SPEC: +4 gutter)
       - `.alphaDecay(0.02)` (per UI-SPEC)
       - `.velocityDecay(0.4)` (per UI-SPEC)
     - Add zoom behavior: `d3.zoom<SVGSVGElement, unknown>().scaleExtent([0.25, 3]).on("zoom", ...)` (per UI-SPEC: 0.25x to 3x)
     - Create `<g>` container inside SVG for zoom transform
     - Render edges as `<line>` elements (z-order: edges drawn first):
       - stroke: `--primary` (#18181b) at opacity 0.25 for curiosity_link, `--color-bridge` at 0.5 for bridge, `--color-misconception` at 0.4 for misconception_cluster (per UI-SPEC Edge Colors)
     - Render nodes as `<g>` groups, each containing:
       - Transparent hit circle: `radius + 10` for 44px minimum touch target (per UI-SPEC: `min(radius + 10, 22)`)
       - Visible circle: fill from `getNodeColor()`, radius from `getNodeRadius()`
       - Text label: 12px Inter 400, `fill: currentColor`, centered below node (per UI-SPEC: 12px exception for SVG labels)
     - Node groups: `role="button"`, `tabIndex={0}`, `aria-label="{concept name}, {health state} concept, {N} visits. Press to view details."` (per UI-SPEC accessibility)
     - Click handler on node groups: call `onNodeClick(node.id)`
     - Hover handler (desktop): show tooltip div at mouse position after 200ms delay (per UI-SPEC: 200ms), content: `"{concept} . {N} visit{s}"`, disappears on mouseout
     - Drag behavior: `d3.drag<SVGGElement, SimNode>()` — dragging sets `fx`/`fy` (pins node), double-click releases (`fx = null, fy = null`)
     - Simulation tick: update line positions (x1, y1, x2, y2) and node group transforms
     - `prefers-reduced-motion` check (per UI-SPEC accessibility): if `window.matchMedia('(prefers-reduced-motion: reduce)').matches`, set `alphaDecay(0.9)` for instant layout
     - CRITICAL cleanup: `return () => { simulation.stop(); svg.selectAll("*").remove(); }` (per RESEARCH pitfall 3)
   - Render: `<div className="relative w-full" style={{ height: "calc(100vh - 120px)" }}>` containing:

     - Tooltip div: `position: absolute; pointer-events: none` with `role="tooltip"` (per UI-SPEC)

| T-03-09 | Information Disclosure | node-detail-panel.tsx | mitigate | Only renders data returned by server action (already access-controlled); no client-side data leakage |
| T-03-10 | Tampering | D3 SVG | accept | Client-side visualization; no security impact if user manipulates their own SVG DOM |
</threat_model>
