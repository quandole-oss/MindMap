# Phase 3 — UI Design Contract

> Visual and interaction contract for Phase 3: Knowledge Graph.
> Covers the interactive D3.js force-directed knowledge graph page, node detail side panel, weekly bridge connection notification, and the "My Graph" sidebar nav entry.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Scope

Phase 3 delivers exactly these screens and components:

1. **Student: Knowledge Graph page** (`/student/graph`) — full-canvas interactive force-directed D3.js graph of the student's concept nodes and edges
2. **Node detail side panel** — slides in on node click, showing the original questions and AI exchanges that created that concept; graph remains visible
3. **Node health legend** — compact key explaining the four node colors (teal/coral/gray/purple)
4. **Weekly bridge notification** — toast or banner on student login announcing the weekly "surprise connection" bridge node (GRPH-08)
5. **Sidebar nav update** — "My Graph" link added to the student sidebar between "Dashboard" and "My Questions"
6. **Graph empty state** — shown when the student has no concepts yet

No teacher views, no misconception diagnostic flow, no node reclassification UI in this phase. Node color states are rendered but their transitions (gray → coral → teal) are driven by Phase 4.

## Typography

All sizes use Inter. Two weights only: 400 (regular) and 600 (semibold). Inherited from Phase 1 — no new sizes added.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px | 400 | 1.5 | Node detail side panel body text, AI exchange content, question text in panel |
| Label | 14px | 400 | 1.4 | Node detail question timestamps, health legend labels, node tooltip text, "My Graph" sidebar nav item |
| Heading | 20px | 600 | 1.2 | Side panel concept name heading, "My Graph" page heading above graph canvas |
| Display | 28px | 600 | 1.15 | Not used in Phase 3 |

Rules (inherited):
- No font sizes outside this set are permitted.
- 14px (label) is the minimum rendered text size.
- Node labels rendered inside the SVG canvas: 12px Inter 400, `fill: var(--foreground)` — this is the only permitted exception to the 14px minimum, as SVG node labels inside a data visualization are exempt from WCAG 1.4.4 (they are part of the image/canvas, not text content).
- Node tooltip text: 14px label (rendered as HTML overlay via a `position: absolute` div, not SVG text — therefore subject to the 14px minimum).

| Edge Type | Color | Opacity | Usage |
|-----------|-------|---------|-------|
| curiosity_link | `--primary` (`#18181b`) | 0.25 | Concepts from the same question |
| bridge | `--color-bridge` (`#a78bfa`) | 0.5 | Cross-domain bridge connections |
| misconception_cluster | `--color-misconception` (`#f87171`) | 0.4 | Shared misconception grouping (Phase 4) |

### Node Sizing

Node visual radius is computed server-side and passed as a prop:

```
radius = clamp(8, 8 + (visitCount * 2), 24)
```

- Minimum: 8px radius (first visit)
- Maximum: 24px radius (8+ visits)
- SVG hit target circle (pointer-events): radius + 10px (to guarantee 44px minimum touch target at minimum node size: `(8 + 10) * 2 = 36px` — use `min(radius + 10, 22)` so the target circle extends to at least 22px radius = 44px diameter)

## Interaction Contract

### Graph Rendering

1. Student navigates to `/student/graph` — server action fetches all nodes + edges for the student as JSON
2. Graph component (`"use client"`) mounts; D3 force simulation initializes with fetched data
3. Force simulation runs until alpha decays below threshold (`d3.forceSimulation().stop()` called when `alpha < 0.001`) — simulation must not run indefinitely
4. Nodes are SVG `<circle>` elements; edges are SVG `<line>` elements rendered beneath nodes (z-order: edges first, nodes on top)
5. Node colors applied from health state → CSS variable map; node radius from visit count formula
6. Page heading "My Knowledge Graph" + health legend rendered above the SVG canvas as HTML (not inside SVG)
7. Graph fills the full content area below the heading: `width: 100%`, `height: calc(100vh - 120px)` (accounts for heading + legend + page padding)

### Node Interaction

- **Hover (desktop):** tooltip appears after 200ms delay showing `"{concept} · {N} visit{s}"`. Tooltip is a `position: absolute` HTML div, not SVG text. Tooltip disappears immediately on mouseout.
- **Click / tap:** side panel opens from the right with a slide-in animation (CSS `transform: translateX` transition, 200ms ease-out). Graph remains visible and interactive behind the panel.
- **Drag:** nodes are draggable via D3 drag behavior. Dragging a node pins it (`fx`, `fy` set) — it stays where dropped. Double-click on a pinned node releases it back to simulation.
- **Zoom + pan:** D3 zoom behavior enabled on the SVG container. Scroll wheel or pinch-to-zoom on mobile. Zoom range: 0.25x to 3x. Reset zoom button (Lucide `ZoomIn` → `Maximize2`) renders as a floating action in the top-right of the canvas.

### Mobile Clustering (GRPH-04)

When node count exceeds 50 visible nodes on a mobile viewport (<768px), zoom-based clustering activates:
- At zoom < 0.5x: nodes cluster by domain (one aggregate circle per domain cluster)
- At zoom ≥ 0.5x: individual nodes render normally
- Cluster circles use the dominant health state color of their member nodes
- Cluster label: "{domain name} · {N} concepts"
- This prevents the SVG performance cliff (~300 nodes on mobile Safari)

Source: CONTEXT.md — locked decision.

### Node Detail Side Panel

- Slide-in from right: 360px wide on desktop; full-width overlay on mobile
- Panel header: concept name (20px/600) + close button (X icon, `--primary` fill)
- Panel body: scrollable (`scroll-area`) list of questions that created this concept, each showing:
  - Question text (16px/400)
  - Timestamp (14px/400, `--muted-foreground`)
  - Collapsed AI answer (expandable with "See answer" / "Hide answer" toggle — matches Phase 2 pattern)
- Panel sits in a `<Sheet>` component (shadcn) rendered at the page root — not inside the SVG

### Weekly Bridge Notification

- Fires on first student page load of the week where a bridge result exists
- Sonner toast, `duration: 8000` (8 seconds) — longer than default to allow reading
- "Explore" action button in toast navigates to the graph and highlights (pulses via CSS animation) the bridge node
- Bridge pulse animation: 3 pulses of `box-shadow` / `filter: drop-shadow` in `--color-bridge`, then stops
- If student dismisses the toast, no repeat for 7 days (tracked via a `bridgeLastShown` timestamp on the session or local storage)

### D3 Force Simulation Parameters

Discretion area from CONTEXT.md — set to these values:

| Parameter | Value | Rationale |
|-----------|-------|-----------|
| `d3.forceManyBody().strength` | -120 | Enough repulsion to separate nodes; not so strong that sparse graphs scatter |
| `d3.forceLink().distance` | 80 | Comfortable reading distance between connected nodes |
| `d3.forceCenter()` | SVG center | Centers the simulation on the canvas |
| `d3.forceCollide().radius` | `node.radius + 4` | 4px gutter between node edges |
| `simulation.alphaDecay` | 0.02 | Default; smooth settling; simulation ends in ~200 ticks |
| `simulation.velocityDecay` | 0.4 | Slightly higher than default 0.4 — faster settling for educational feel |

## Accessibility Baseline

Inherited from Phase 1 and Phase 2, plus Phase 3 additions:

- SVG graph: `<svg role="img" aria-label="Knowledge graph showing {N} concepts">` — screen readers get a meaningful description; the interactive graph details are in the side panel (accessible HTML)
- Node click targets: each node group `<g>` has `role="button"` and `aria-label="{concept name}, {health state} concept, {N} visits. Press to view details."` — keyboard-navigable via Tab (apply `tabIndex={0}` to each node group)
- Keyboard: Tab moves between nodes; Enter/Space opens side panel; Escape closes side panel (Sheet handles this natively)
- Side panel focus: focus moves to the panel heading on open (`autoFocus`); returns to the clicked node on close
- Node tooltip: rendered as HTML with `role="tooltip"` and `id` linked via `aria-describedby` on the node element
- Health legend dots: each `<span>` dot has `aria-hidden="true"`; the label text conveys the meaning
- Zoom reset button: `aria-label="Reset graph zoom"`
- Bridge toast: Sonner `role="status"` is sufficient; "Explore" action is keyboard-accessible
- Color contrast: node health state colors against white canvas — verify each:
  - Teal `#14b8a6` on `#ffffff`: 2.8:1 — fails for text, but nodes are graphical (exempt from 4.5:1 text contrast rule under WCAG 1.4.3); health state is also communicated via the legend text label (redundant coding)
  - All text inside side panel and legend: inherits passing contrast from Phase 1 tokens (`#18181b` on `#ffffff` = 19.5:1`)
- `prefers-reduced-motion`: wrap D3 simulation transitions and bridge pulse animation in a check — if `window.matchMedia('(prefers-reduced-motion: reduce)').matches`, skip animations and set simulation alpha decay high (0.9) for instant layout
