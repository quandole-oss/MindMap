# Phase 1 — UI Design Contract

> Visual and interaction contract for Phase 1: Foundation.
> Covers landing page, authentication screens (sign-up, log-in), and the authenticated app shell (sidebar nav, class creation/join, class roster).
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Scope

Phase 1 delivers exactly these screens:

1. **Landing page** — hero section + sign-up / log-in entry points
2. **Sign-up screen** — email, password, student/teacher role toggle
3. **Log-in screen** — email, password
4. **Teacher: Create class** — class name input, join code display
5. **Teacher: Class roster** — student list with grade level, join code badge
6. **Student: Join class** — join code input
7. **Student: Profile stub** — grade level display (set by teacher)

No graph views, no curiosity engine, no misconception UI in this phase.

## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, tight inline padding |
| sm | 8px | Input field internal padding (top/bottom), badge padding |
| md | 16px | Default element spacing, form field gaps, button padding |
| lg | 24px | Card internal padding, section breaks within a form |
| xl | 32px | Layout column gaps, vertical section separation |
| 2xl | 48px | Hero section vertical padding, page-level content margins |
| 3xl | 64px | Landing hero full-bleed top/bottom padding |

Exceptions:
- Touch targets for all interactive controls (buttons, inputs): minimum 44px height — satisfies WCAG 2.5.5 AA
- Sidebar nav item height: 40px (compact desktop, not a touch-primary surface)

## Typography

All sizes use Inter. Two weights only: 400 (regular) and 600 (semibold).

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px | 400 | 1.5 | Form labels, helper text, roster table cells, error messages |
| Label | 14px | 400 | 1.4 | Input labels, sidebar nav items, table column headers, badge text |
| Heading | 20px | 600 | 1.2 | Page headings (Sign Up, Class Roster, Create Class), card titles |
| Display | 28px | 600 | 1.15 | Landing hero headline only |

Rules:
- No other font sizes are permitted in Phase 1.
- 14px (label) is the minimum — never go below for any rendered text.
- Hero subheadline (landing page tagline) uses Body 16px at weight 400.

## Color

Light mode only. No dark mode tokens in Phase 1.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#ffffff` (white) | Page backgrounds, form backgrounds, modal/dialog backgrounds |
| Secondary (30%) | `#f4f4f5` (zinc-100) | Sidebar background, card backgrounds, table row alternating fill, input backgrounds |
| Accent (10%) | `#18181b` (zinc-900) | See reserved list below |
| Destructive | `#ef4444` (red-500) | Error messages, form validation errors — no other use |

Accent (`#18181b`) reserved for:
1. Primary CTA buttons (Sign Up, Log In, Create Class, Join Class, Save)
2. Active sidebar nav item indicator (left border or background highlight)
3. Join code badge background on the roster view

All other interactive elements (secondary buttons, links, hover states) use secondary palette. No accent color on decorative elements, icons, or borders.

Semantic additions for node health states (defined here for consistency — used in Phase 3 but tokens established now):
| State | Value | Meaning |
|-------|-------|---------|
| Healthy | `#14b8a6` (teal-500) | Resolved/healthy concept node |
| Misconception | `#f87171` (red-400) | Coral — active misconception node |
| Unprobed | `#a1a1aa` (zinc-400) | Gray — not yet diagnosed |
| Bridge | `#a78bfa` (violet-400) | Cross-subject bridge node |

These tokens are declared in Phase 1 CSS but rendered only in Phase 3.

## Layout Contract

### Landing Page

- Full-width, centered content column, max-width 640px
- Hero: Display headline + Body subline + stacked CTAs ("Get Started" primary, "Log In" ghost/outline)
- No sidebar on landing page
- No navigation bar — just the MindMap wordmark top-left

### Auth Screens (Sign-up, Log-in)

- Centered card layout, card width 400px, on white background
- Card uses secondary surface (`#f4f4f5`) with 24px padding
- No sidebar — full-width background
- MindMap wordmark top-center above card

### Authenticated Shell (teacher and student)

- Left sidebar: 240px fixed width, secondary surface (`#f4f4f5`)
- Content area: remaining width, white background, 32px horizontal padding
- Sidebar nav items: 40px height, 16px horizontal padding, 14px label text
- Page heading: 20px Heading, 32px top margin, 24px bottom margin before first content block
- No top navigation bar — sidebar handles all nav
- Mobile: sidebar collapses to icon-only (48px) or off-canvas at <768px breakpoint

## Accessibility Baseline

All Phase 1 screens must meet WCAG AA at minimum:

- Color contrast: all body text on backgrounds must meet 4.5:1 ratio (`#18181b` on `#ffffff` = 19.5:1, passes)
- Focus rings: shadcn default focus-visible rings are sufficient — do not remove them
- Form errors: error messages are associated with inputs via `aria-describedby`
- Role toggle: `toggle-group` must be wrapped in a `fieldset` with a visible `legend` ("I am a...")
- Destructive dialog: focus moves to `AlertDialog` on open; returns to trigger on close
