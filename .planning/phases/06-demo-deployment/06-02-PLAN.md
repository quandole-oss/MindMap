**3. Create `Dockerfile`** at repo root using multi-stage build with turbo prune:

Stage 1 — `base`: `node:20-alpine` with pnpm enabled via corepack
Stage 2 — `pruner`: Copy full repo, run `turbo prune web --docker` to generate minimal workspace
Stage 3 — `installer`: Copy pruned `json/` (package.jsons only), run `pnpm install --frozen-lockfile`. Then copy pruned `full/` (all source), run `pnpm turbo run build --filter=web`
Stage 4 — `runner`: `node:20-alpine` production image. Copy `apps/web/.next/standalone` and `apps/web/.next/static` and `apps/web/public` (if exists). Set `HOSTNAME=0.0.0.0`, `PORT=3000`. `CMD ["node", "apps/web/server.js"]`

Key details:
- Enable corepack in base: `RUN corepack enable && corepack prepare pnpm@9 --activate`
- In pruner: `COPY . .` then `RUN pnpm turbo prune web --docker`
- In installer: Copy from pruner `/app/out/json/` (lockfile + package.jsons), then `pnpm install --frozen-lockfile`
- Then copy from pruner `/app/out/full/` and run `pnpm turbo run build --filter=web`
- In runner: Copy standalone output, set env vars, expose port 3000
- The standalone server.js path will be at `apps/web/server.js` relative to the standalone output root
- Copy static assets: `COPY --from=installer /app/apps/web/.next/static ./apps/web/.next/static`
- Copy public if it exists: check first with `ls apps/web/public` — if it doesn't exist, skip this COPY

**IMPORTANT for turbo prune --docker**:
- The `out/json/` directory contains only package.json files and lockfile
- The `out/full/` directory contains the full pruned source
- This ensures Docker layer caching is effective (dependencies only reinstall when package.json changes)

**4. Update `docker-compose.yml`** to add the web service:

```yaml
services:
  postgres:
    image: pgvector/pgvector:pg16
    ports:
      - "5432:5432"
    environment:
      POSTGRES_USER: mindmap
      POSTGRES_PASSWORD: mindmap
      POSTGRES_DB: mindmap
    volumes:
      - pgdata:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U mindmap"]
      interval: 5s
      timeout: 5s
      retries: 5

  web:
    build:
      context: .
      dockerfile: Dockerfile
    ports:
      - "3000:3000"
    environment:
      DATABASE_URL: postgresql://mindmap:mindmap@postgres:5432/mindmap
      AUTH_SECRET: ${AUTH_SECRET:-demo-secret-change-in-production-min-32-chars}
      ANTHROPIC_API_KEY: ${ANTHROPIC_API_KEY:-}
      OPENAI_API_KEY: ${OPENAI_API_KEY:-}
      NODE_ENV: production
      NEXT_TELEMETRY_DISABLED: "1"
    depends_on:
      postgres:
        condition: service_healthy
    restart: unless-stopped

volumes:
  pgdata:
```

Note: The web service uses the postgres healthcheck to wait for the database before starting. Environment variables pass through from host `.env` or use defaults. `NEXT_TELEMETRY_DISABLED=1` enforces INFR-05.
