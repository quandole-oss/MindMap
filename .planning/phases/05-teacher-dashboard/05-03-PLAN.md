**Step 1: Create `apps/web/components/dashboard/concepts-tab.tsx`** (no `"use client"` needed — pure props).

Props: `{ heatmap: ConceptHeatmapEntry[] }`.

Render a table using shadcn Table component with columns:

| Column | Content |
|--------|---------|
| Concept | `entry.conceptName` |
| Domain | `<Badge variant="secondary">{entry.domain}</Badge>` |
| Students | `entry.studentCount` — also render a background color on the entire row |
| Total Visits | `entry.totalVisits` |

**Heatmap color encoding**: The `studentCount` determines row background color intensity. Compute the max studentCount in the dataset. For each row, compute `intensity = studentCount / maxStudentCount`. Apply as inline style: `backgroundColor: \`rgba(20, 184, 166, ${0.05 + intensity * 0.2})\`` (teal with 5-25% opacity). This creates a subtle gradient from near-white to light teal for the most common concepts.

Sort entries by `studentCount` descending (most widespread first).

If `heatmap.length === 0`, render empty state: "No concepts recorded yet across this class."

Export as named `ConceptsTab`.

**Step 2: Create `apps/web/components/dashboard/misconceptions-tab.tsx`** (`"use client"` — needs useState for expand/collapse of student names).

Props: `{ clusters: MisconceptionCluster[] }`.

Render misconception cluster cards using the existing Card component in a `grid grid-cols-1 md:grid-cols-2 gap-4` layout.

Each card contains:
1. **Header**: `cluster.misconceptionName` as card title (text-[16px] font-semibold)
2. **Stats row**: `{cluster.totalAffected} students affected` as text, with `{cluster.resolvedCount} resolved` in teal and `{cluster.unresolvedCount} unresolved` in coral
3. **Progress bar**: A horizontal bar showing repair progress.
   - Container: `h-2 w-full bg-[#f4f4f5] rounded-full overflow-hidden`
   - Fill: `h-full bg-[#14b8a6] rounded-full` with `width: ${(cluster.resolvedCount / cluster.totalAffected) * 100}%`
   - Handle divide-by-zero (totalAffected=0 → 0%)
   - Label below: `${Math.round((cluster.resolvedCount / cluster.totalAffected) * 100)}% resolved`
4. **Drill-down**: A disclosure toggle ("Show affected students" / "Hide") that expands to show `cluster.affectedStudentNames` as a comma-separated list. Use `useState<string | null>` to track which cluster ID is expanded.

Sort clusters by `unresolvedCount` descending (most urgent first).

If `clusters.length === 0`, render empty state: "No misconceptions detected in this class yet."

Export as named `MisconceptionsTab`.
