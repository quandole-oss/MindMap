**Step 2: Create `apps/web/actions/dashboard.ts`** as a `"use server"` module exporting `getClassDashboardData(classId: string)`.

Implementation details:

1. **Auth + ownership check**: Call `auth()`, verify `session.user.id` exists. Query `classes` table with `and(eq(classes.id, classId), eq(classes.teacherId, session.user.id))`. Return `{ error: "Not authorized" }` if not found. This follows the exact pattern from `getClassRoster` in `actions/class.ts`.

2. **Get enrolled students**: Query `classEnrollments` joined with `users` where `classId` matches. Extract `studentIds` array.

3. **Per-student concept data**: For each studentId, query `concepts` table (`where: eq(concepts.userId, studentId)`) getting id, name, domain, status, visitCount. Also query `conceptEdges` where sourceConceptId or targetConceptId is in that student's concept IDs (same pattern as `getGraphData` in `actions/graph.ts` but without bridge detection). Bundle into `StudentGraphData`.

4. **Per-student question data**: Query `questions` table per student to get totalQuestions and lastQuestionDate (use `desc(questions.createdAt)` with `limit: 1` for last date). For streak calculation: query questions ordered by createdAt desc, iterate backwards counting consecutive days with at least one question (same logic as existing streak in the student dashboard — count from today/yesterday backward).

5. **Breadth score**: Collect all unique domains from the student's concepts (`new Set(concepts.map(c => c.domain))`). For `totalDomains`, collect all unique domains across ALL students in the class (compute once, share). `breadthScore = uniqueDomains / totalDomains` (handle divide-by-zero → 0).

6. **Inactivity check**: `isInactive = lastQuestionDate === null || (Date.now() - lastQuestionDate.getTime()) > 3 * 24 * 60 * 60 * 1000` (3+ days).

7. **Concept heatmap**: After collecting all per-student concepts, aggregate by concept name (case-insensitive grouping via `.toLowerCase()`). For each unique concept name, count distinct students and sum visitCounts. Sort by studentCount descending.

8. **Misconception clusters**: Query `diagnosticSessions` where `userId` is in `studentIds`. Group by `misconceptionId`. For each group: count total unique userIds (totalAffected), count where `outcome = 'resolved'` (resolvedCount), remainder is unresolvedCount. Join user names for `affectedStudentNames`.

9. **Totals**: `totalStudents`, `totalQuestions` (sum across students), `activeMisconceptions` (count of misconception clusters where unresolvedCount > 0), `avgBreadthScore` (average of all student breadthScores).

10. Return `ClassDashboardData` typed object.

**Important**: Use `inArray` for batch queries where possible rather than N+1 loops. For concepts, do ONE query `where: inArray(concepts.userId, studentIds)` then partition by userId in JS. Same for questions. This keeps it to ~5 total DB queries, not O(N) per student.

**Important**: Import from `drizzle-orm`: `eq, and, inArray, or, desc, count, sql`. Import `db, schema` from `@mindmap/db`. Import `auth` from `@/lib/auth`.

**Component signature:**
```typescript
interface MiniGraphSvgProps {
  nodes: Array<{ id: string; name: string; domain: string; status: "unprobed" | "healthy" | "misconception"; visitCount: number }>;
  edges: Array<{ source: string; target: string }>;
  width?: number;   // default 160
  height?: number;  // default 120
  className?: string;
}
```

**Implementation:**

1. **Layout**: Use a simple force-free layout. Position nodes in a circle within the SVG viewBox (center at width/2, height/2, radius = min(width, height) * 0.35). If 0 nodes, render empty SVG with a subtle "No data" text.

2. **Node coloring**: Match the existing knowledge-graph.tsx color scheme:
   - `healthy` → `#14b8a6` (teal)
   - `misconception` → `#f87171` (coral)
   - `unprobed` → `#a1a1aa` (gray)

3. **Node sizing**: Base radius = 4px. Scale slightly by visitCount: `radius = 4 + Math.min(visitCount, 10) * 0.3`. Cap at ~7px to keep thumbnails clean.

4. **Edges**: Render as `<line>` elements between node positions, `stroke="#e4e4e7"` (light gray), `strokeWidth={0.5}`.

5. **No labels**: Thumbnails are too small for text. Only circles and lines.

6. **No "use client"**: This component is a pure function of props, no hooks, no state, no D3. It renders a plain `<svg>` element. This means it can render on the server (RSC-compatible).

7. **Accessible**: Add `role="img"` and `aria-label="{N} concepts"` to the SVG element.
