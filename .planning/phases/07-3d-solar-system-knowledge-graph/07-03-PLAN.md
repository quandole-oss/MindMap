1. LOD in solar-nodes.tsx (per D-15):
   Add a second InstancedMesh layer for distant points:
   - Import `useFrame, useThree` from `@react-three/fiber`
   - Track camera position via `useFrame` — on each frame, calculate distance from camera to each node
   - LOD approach: Use TWO InstancedMesh instances sharing the same node data:
     a. "spheres" InstancedMesh: `<sphereGeometry args={[1, 16, 16]} />` — detailed spheres
     b. "points" InstancedMesh: `<sphereGeometry args={[1, 4, 4]} />` — low-poly stand-in (visually reads as a point at distance)
   - In a `useFrame` callback: for each node, check `camera.position.distanceTo(nodePosition)`. If distance > 150, scale the sphere instance to 0 (hide) and scale the point instance to normal size. If distance <= 150, do the reverse.
   - Implementation detail: create a `Vector3` in a ref for distance calculations to avoid allocations in useFrame
   - Both InstancedMeshes share the same material properties (toneMapped={false}, vertexColors, emissive)
   - The "points" mesh uses a smaller sphere geometry (args [1, 4, 4]) that at distance renders as a dot

   Alternative simpler approach (if the two-mesh approach is too complex): Use a single InstancedMesh but in useFrame, dynamically set scale per-instance based on distance — close nodes get full scale, distant nodes get 0.3x scale. This achieves visual LOD without geometry switching:
   ```typescript
   useFrame(({ camera }) => {
     if (!meshRef.current) return;
     const camPos = camera.position;
     layoutNodes.forEach((node, i) => {
       const dist = camPos.distanceTo(tempVec.set(node.x, node.y, node.z));
       const baseRadius = getNodeRadius(node);
       const lodScale = dist > 150 ? 0.3 : 1;
       dummy.position.set(node.x, node.y, node.z);
       dummy.scale.setScalar(baseRadius * lodScale);
       dummy.updateMatrix();
       meshRef.current.setMatrixAt(i, dummy.matrix);
     });
     meshRef.current.instanceMatrix.needsUpdate = true;
   });
   ```
   Use this simpler approach — it's one InstancedMesh, achieves the LOD visual effect (distant nodes appear as tiny dots), and avoids the complexity of managing two meshes.
