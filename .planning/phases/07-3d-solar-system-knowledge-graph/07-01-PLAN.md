   - Export a `LayoutNode` type extending `GraphNode` with `x: number; y: number; z: number`
   - Export a `useGraphLayout(nodes: GraphNode[], edges: GraphEdge[]): LayoutNode[]` hook
   - Inside the hook, use `useMemo` keyed on `[nodes, edges]`
   - In the memo: deep-clone nodes via `nodes.map(n => ({ ...n }))` and edges via `edges.map(e => ({ ...e }))` — d3-force-3d mutates objects in-place (per D-02, same lesson from 2D graph)
   - Import from `d3-force-3d` specifically (NOT from `d3`): `forceSimulation, forceLink, forceManyBody, forceCenter, forceCollide`
   - Create simulation: `forceSimulation(simNodes, 3)` — the second arg `3` sets numDimensions to 3
   - Add forces: `.force("link", forceLink(simLinks).id((d: any) => d.id).distance(60))`, `.force("charge", forceManyBody().strength(-180))`, `.force("center", forceCenter(0, 0, 0))`, `.force("collide", forceCollide().radius(14))`
   - Run synchronously: `.stop()` then `.tick(300)` — this produces converged positions without triggering React re-renders
   - Return `simNodes` cast as `LayoutNode[]`
   - Add `"use client"` directive at top since this uses React hooks
   - Handle empty nodes array: return `[]` immediately if `nodes.length === 0`

Note: Import exclusively from `d3-force-3d`, never from `d3` in this file, to avoid TypeScript namespace conflicts with the existing `@types/d3` package (per Research Pitfall 7).
