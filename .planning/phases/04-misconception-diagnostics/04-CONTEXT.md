# Phase 4: Misconception Diagnostics - Context

**Gathered:** 2026-04-08
**Status:** Ready for planning

<domain>
## Phase Boundary

Implement the complete Socratic diagnostic session flow: when the routing engine selects diagnose mode, the student enters a multi-turn conversation that probes their mental model, classifies the misconception, generates a targeted confrontation scenario, and resolves (or not). Node health states update to reflect outcomes. Students see the named misconception after the session.

</domain>

<decisions>
## Implementation Decisions

### Diagnostic Session Flow
- Database-backed session state: diagnostic_sessions table with stage enum (probe/classify/confront/resolve) — survives browser crashes and page reloads
- Automatic entry: when routing picks diagnose, /api/ask starts a diagnostic session instead of the normal enrich answer
- Chat-like multi-turn UI within the question page — each exchange is a back-and-forth between student and AI
- Node turns coral when session starts (misconception detected), teal when student demonstrates understanding in resolve stage

### Misconception Reveal & Resolution
- Confrontation scenarios generated by LLM using misconception library entry's confrontation_scenarios as seed — personalized counterexample
- Resolution detection: after confrontation, student responds; LLM evaluates if understanding shifted — marks resolved or unresolved
- Named misconception revealed after full session (resolve stage): "You were thinking about X — this is called the Y misconception"
- Abandoned sessions: node stays coral, session marked "incomplete" — can resume later

### Claude's Discretion
- Specific prompt templates for each diagnostic stage (probe, classify, confront, resolve)
- diagnostic_sessions table schema details
- Multi-turn streaming conversation implementation
- Resolution evaluation criteria and prompt
- UI layout for diagnostic conversation vs normal Q&A

</decisions>

<specifics>
## Specific Ideas

- The 4-stage flow: probe → classify → confront → resolve maps to the diagnostic_sessions.stage column
- Probe: AI asks open-ended question from misconception library's probe_questions
- Classify: AI analyzes student response against known misconception patterns
- Confront: AI generates targeted counterexample using confrontation_scenarios as seed
- Resolve: AI evaluates if student's understanding shifted; updates node health state
- Each stage is a streaming LLM response followed by student text input
- The full conversation history is maintained in the session for context

</specifics>
