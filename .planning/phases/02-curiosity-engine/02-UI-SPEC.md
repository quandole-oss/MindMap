# Phase 2 — UI Design Contract

> Visual and interaction contract for Phase 2: Curiosity Engine.
> Covers the student curiosity question submission flow, streaming AI answer display, Socratic follow-up, streak counter, question history, and the post-submission concept node confirmation.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Scope

Phase 2 delivers exactly these screens and components:

1. **Student dashboard — question panel** — centered question input with "What are you curious about today?" prompt, submit button, and already-asked-today state
2. **Streaming answer view** — inline markdown answer rendering below the question with skeleton loader while streaming
3. **Socratic follow-up display** — follow-up question rendered as a distinct callout after the answer
4. **Streak badge** — days-in-a-row counter displayed near the student's name in the sidebar/header
5. **Question history page** — reverse-chronological list with date headers, each entry expandable to show the AI response
6. **Concept extraction confirmation** — small toast/notice after submission: "X new concepts added to your graph"
7. **Daily limit state** — blocked state shown when the student has already submitted today

No D3 graph rendering, no misconception diagnostic flow, no teacher views in this phase.

## Typography

All sizes use Inter. Two weights only: 400 (regular) and 600 (semibold). Inherited from Phase 1 — no new sizes added.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px | 400 | 1.5 | AI answer body text, history entry body, helper text, streak label text |
| Label | 14px | 400 | 1.4 | Date headers in history, concept tag labels, "already asked today" notice, toast messages |
| Heading | 20px | 600 | 1.2 | "My Questions" page heading, question text in history cards |
| Display | 28px | 600 | 1.15 | Not used in Phase 2 |

Rules (inherited):
- No font sizes outside this set are permitted.
- 14px (label) is the minimum rendered text size.
- AI answer markdown renders body (16px/400) for paragraphs, heading (20px/600) for any h2/h3 the LLM may produce.
- Socratic follow-up question uses body 16px at weight 600 (semibold) to visually distinguish it from the answer body.

## Interaction Contract

### Question Submission Flow

1. Student lands on dashboard — question panel is the visual focus (centered, top of content area)
2. Student types in textarea — no character limit enforced (REQUIREMENTS.md does not specify one); textarea expands to 3-line min
3. Student submits — submit button shows loading spinner (Lucide `Loader2` with `animate-spin`), textarea disabled
4. Page transitions to answer view inline below the question panel — no full-page navigation
5. Answer streams character-by-character via Vercel AI SDK streaming — skeleton shows for first 300ms before characters arrive
6. Socratic follow-up question appears below the streamed answer separated by a horizontal rule
7. Concept toast fires after streaming completes (from server action response, not the stream)
8. Question panel replaced with the daily-limit state for the rest of the day

### Answer Markdown Rendering

- Use `react-markdown` (Claude's discretion from CONTEXT.md)
- Strip any `<script>` or raw HTML from LLM output — render text/markdown only
- Do not render images from LLM output (no `img` elements)
- Code blocks: render in `<pre>` with `--muted` background, monospace font, 14px — grade school topics may reference code examples

### Streaming Skeleton

- Show a pulsing skeleton (shadcn `skeleton`) with 3 lines (simulating body text) for the first 300ms
- Replace skeleton with the streaming text as soon as first characters arrive
- No hard-coded min render time — start showing characters immediately after skeleton grace period

### History List

- Reverse-chronological: newest entries at top
- Date header sticky on scroll within the list (CSS `position: sticky`)
- Each entry: question text (heading 20px/600) + timestamp (label 14px/400) + collapsed answer
- "See answer" toggle expands inline — no page navigation
- No pagination in Phase 2 — render up to 30 entries; note in code that Phase 3+ may need virtualization

### Streak Badge Placement

- Rendered in the sidebar below the student's name — not in the main content area
- Uses secondary surface with `Flame` icon and count in label (14px/400)
- Updates on next page load after a new question is submitted (server-rendered, no real-time sync needed)

- Inherited shell: 240px left sidebar + remaining content area with 32px horizontal padding

- Textarea: full column width, min-height 88px, `--border` border, `--muted` background, `--radius-lg` border-radius
- Submit button: right-aligned below textarea, height 44px, accent background (`--primary`)

- Rendered as a card (`--card` background) with 24px padding directly below the question panel, 16px gap
- Streamed answer: full card width, body text
- Separator: `<Separator>` component, 16px vertical margin above and below
- Socratic follow-up: `--muted` background block, 16px padding, 2px left border in `--primary`, inside the card

### Daily Limit State

- Replaces question panel — same visual position
- Uses `--muted` background card with "You've asked your question for today" heading
- Today's submitted question and answer still render below in the same answer view card

### Question History Page

- Route: `/student/questions`
- Same shell layout as dashboard
- Content column: full-width scroll area, no max-width constraint (list needs full width)
- Date headers: sticky, `--muted` background, label (14px/400), 8px vertical padding
- History entries: card per entry, 16px gap between cards

### Mobile (inherited from Phase 1)

- Sidebar collapses at <768px breakpoint — same behavior as Phase 1
- Question panel: full viewport width with 16px horizontal padding
- No horizontal scrolling on answer view — markdown wraps naturally

## Accessibility Baseline

Inherited from Phase 1 plus Phase 2 additions:

- Streaming answer region: wrap in `<div role="status" aria-live="polite" aria-label="Loading answer">` — screen readers announce when streaming completes
- Textarea: associated `<label>` via `htmlFor`, even if the label is visually hidden (`sr-only`) when the placeholder is visible
- Streak badge: `aria-label="Current streak: {N} days"` on the badge container
- Toast (Sonner): Sonner uses `role="status"` by default — confirm it is accessible on init
- History expand toggle: `aria-expanded` state toggled on the "See answer" / "Hide answer" button
- Date headers in history: use `<h2>` semantically — they are section headings, not just styled dividers
- Color contrast: all text on `--background` and `--muted` surfaces must maintain 4.5:1 minimum ratio (inherited tokens pass — `#18181b` on `#ffffff` = 19.5:1, `#18181b` on `#f4f4f5` = 17.5:1)
