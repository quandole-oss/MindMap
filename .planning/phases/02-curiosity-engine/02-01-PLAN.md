<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User question -> LLM prompt | Untrusted user text sent to Claude API; prompt injection risk |
| LLM response -> concept extraction | LLM output parsed as structured data; malformed output risk |
| Environment variable -> provider selection | LLM_PROVIDER env controls code path; injection via env unlikely but validated |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Information Disclosure | packages/llm/src/prompts/enrich.ts | mitigate | buildEnrichSystemPrompt uses ONLY gradeLevel (integer) — never name, email, userId. Test enforces no PII patterns in output. |
| T-02-02 | Tampering | packages/llm/src/prompts/extract.ts | mitigate | conceptExtractionSchema validates output with Zod; malformed LLM output rejected at parse. AI SDK Output.object handles retries. |
| T-02-03 | Tampering | User question text in prompt | accept | User question is placed in `prompt:` position (not `system:`). Claude 3.5+ is resilient to simple prompt injection. System prompt is authoritative. Low risk for educational K-12 context. |
| T-02-04 | Elevation of Privilege | packages/llm/src/adapters/factory.ts | mitigate | Factory validates LLM_PROVIDER against allowlist; unknown values throw immediately rather than instantiating arbitrary code. |
</threat_model>
