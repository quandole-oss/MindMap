    `getQuestionHistory()`:
    - Auth check via `auth()`
    - Query: `db.select().from(schema.questions).where(eq(schema.questions.userId, userId)).orderBy(desc(schema.questions.createdAt)).limit(30)`
    - Return array of question rows (limit 30 per UI-SPEC: "render up to 30 entries")

    `getStreak()`:
    - Auth check via `auth()`
    - Query distinct dates with questions:
      ```typescript
      const rows = await db
        .selectDistinct({ day: sql<string>`DATE(${schema.questions.createdAt} AT TIME ZONE 'UTC')` })
        .from(schema.questions)
        .where(eq(schema.questions.userId, userId))
        .orderBy(desc(sql`DATE(${schema.questions.createdAt} AT TIME ZONE 'UTC')`))
        .limit(365);
      ```
    - Compute streak: iterate rows, compare each to expected consecutive day (starting from today UTC). Break on first non-consecutive day. Return count.
    - If no rows or today has no question, check if yesterday has one (streak doesn't break until the day passes without a question). Actually: if today has no question yet, the streak should still count yesterday's run. Logic: start from today; if today has a question, start counting from today. If not, start from yesterday. Count consecutive days backward.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client -> /student/questions | Authenticated student views their own data only |
| Server action -> Database | Questions filtered by session.user.id |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-13 | Spoofing | apps/web/actions/questions.ts | mitigate | All query functions check `auth()` session; userId from session only. getQuestionHistory filters by session.user.id — students cannot view other students' questions. |
| T-02-14 | Information Disclosure | apps/web/app/student/questions/page.tsx | mitigate | Server component calls getQuestionHistory which filters by authenticated user. No cross-user data exposure. |
| T-02-15 | Denial of Service | apps/web/actions/questions.ts | accept | getQuestionHistory limited to 30 rows. getStreak limited to 365 rows. No unbounded queries. Acceptable for v1. |
</threat_model>
