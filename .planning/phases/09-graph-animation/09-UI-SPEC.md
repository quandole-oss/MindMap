## Color

This phase operates entirely within the dark WebGL canvas context. All overlay surfaces use the established `bg-black/50 backdrop-blur-md border border-white/10` pattern from the existing graph page.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `oklch(0.145 0 0)` — near-black | WebGL canvas background, page body |
| Secondary (30%) | `rgba(0,0,0,0.5)` with `backdrop-blur-md` | Growth summary card, transition message, HealthLegend container |
| Accent (10%) | `#ffffff` at 10–70% opacity | Text on overlays; particle sparkles (white/near-white) |
| Destructive | `oklch(0.704 0.191 22.216)` | Not used in this phase |

Source: `globals.css` dark mode tokens + established overlay pattern from `graph-page-client.tsx` line 127.

**Accent reserved for:**
- Sparkle particle color for node birth effect (white/near-white `#ffffff`)
- Growth summary count numbers (`+3 concepts, +2 connections` — rendered in full white `text-white`)
- "Your graph is growing..." transition message text (full white)

**Node health state colors (existing — do not change):**

| State | Base hex | Bloom multiplier | CSS variable |
|-------|----------|-----------------|--------------|
| Healthy | `#0d9488` (teal) | 4.0× | `--color-healthy` |
| Misconception | `#dc2626` (coral/red) | 5.0× | `--color-misconception` |
| Unprobed | `#71717a` (gray) | 3.0× | `--color-unprobed` |
| Bridge | `#7c3aed` (purple) | 6.0× | `--color-bridge` |

Source: `solar-nodes.tsx` lines 14–19 + `globals.css` lines 120–124.

**Sparkles particle color:**
- Use `#ffffff` (pure white) with `opacity` fading from 1.0 → 0.0 over the 500ms birth duration. This ensures sparkles do not compete visually with the node's own bloom color.

## Animation Contract

This section is unique to this phase. All animation values come from CONTEXT.md decisions (D-07 through D-11).

### Node Birth (GANIM-02)

| Parameter | Value | Source |
|-----------|-------|--------|
| Scale-up duration per node | 500ms | D-07 |
| Stagger between nodes | 300ms | D-07 |
| Easing | Ease-out cubic (`t = 1 - (1-t)^3`) | Claude's Discretion |
| Starting scale | 0 (invisible) | D-07 |
| Final scale | `getNodeRadius(node)` world units | existing pattern |
| Implementation | `useFrame` loop with `Map<instanceIndex, birthTimestamp>` ref | D-16 |

### Particle Birth Effect (GANIM-05)

| Parameter | Value | Source |
|-----------|-------|--------|
| Component | Drei `<Sparkles>` | D-10 |
| Particle count | 12 per node | Claude's Discretion |
| Particle color | `#ffffff` | Claude's Discretion |
| Particle size | 0.4 world units | Claude's Discretion |
| Effect duration | 500ms (matches node scale-up) | D-10 |
| Fade behavior | Opacity 1.0 → 0.0 over duration | D-10 |

### Edge Draw-In (GANIM-03)

| Parameter | Value | Source |
|-----------|-------|--------|
| Start trigger | After all new nodes have settled (~1s after last node starts) | D-08 |
| Draw-in duration per edge | 400ms | D-08 |
| Easing | Linear (consistent with constellation-line visual metaphor) | Claude's Discretion |
| Implementation | Progress `0 → 1` uniform passed to Line shader or dashOffset technique | D-08 |

### Camera Sequence (GANIM-04)

| Parameter | Value | Source |
|-----------|-------|--------|
| Step 1: Pull back | Frame all new + nearby nodes via bounding sphere + margin | D-09 |
| Step 2: Settle | Camera lerps to final framing position | D-09 |
| Lerp factor | 0.06 per frame | existing `solar-scene.tsx` pattern |
| Trigger | After node animation starts | D-09 |

### Full sequence timeline

```
t=0ms      First new node scale-up begins + sparkles
t=300ms    Second node scale-up begins (if N>1)
t=600ms    Third node scale-up begins (if N>2)
t=1000ms   All nodes settled; edge draw-in begins
t=1400ms   Edges complete; camera settling
t=1500ms   Camera at rest
t=2000ms   Growth summary overlay appears (fade-in 200ms)
t=6000ms   Growth summary fades out (400ms)
```

### Reduced Motion (GANIM-07)

When `window.matchMedia('(prefers-reduced-motion: reduce)').matches` is `true`:
- Skip all scale-up animation — nodes appear at final scale instantly
- Skip sparkles entirely — do not mount `<Sparkles>` components
- Skip edge draw-in — all edges appear at full opacity instantly
- Skip camera lerp — camera jumps directly to final framing position
- Show growth summary overlay immediately (no fade-in delay)
- Growth summary still auto-dismisses after 4s

## Overlay Components

### Transition Message (question-form.tsx → before navigation)

Renders in the answer/question page during the 2s delay before `router.push`.

```
Appearance: centered below the answer content
Background: bg-muted rounded-xl p-4
Text: "Your graph is growing..."
Text style: text-[14px] text-muted-foreground
Icon: Sparkles (lucide, size-4) inline left of text
```

Source: D-02 + existing `question-form.tsx` card pattern (`bg-muted rounded-xl p-6`).

### Growth Summary Overlay (graph canvas)

Renders as an absolutely-positioned card anchored bottom-center, above the HealthLegend.

```
Position: absolute bottom-16 left-1/2 -translate-x-1/2 z-20
           (bottom-16 = 64px, sits above the HealthLegend card at bottom-4)
Background: bg-black/50 backdrop-blur-md border border-white/10 rounded-lg px-5 py-2.5
Heading: "+{N} concepts, +{N} connections"
Heading style: text-[20px] font-semibold text-white
Sub-label: "added to your graph"
Sub-label style: text-[13px] text-white/70
Dismiss: fades out after 4s; immediate dismiss on any graph interaction (click, drag, filter)
Fade-in: opacity 0 → 1 over 200ms (CSS transition)
Fade-out: opacity 1 → 0 over 400ms (CSS transition)
```

Source: D-12, D-13, D-14 + existing overlay pattern from `graph-page-client.tsx` line 127.

### "Preparing your graph..." Polling State

Renders during the concept readiness polling window (max 10s, 5 attempts).

```
Position: same as transition message location in question-form.tsx
Text: "Preparing your graph..."
Style: text-[13px] text-white/40
Icon: Loader2 (lucide, size-3.5, animate-spin, text-white/40)
Layout: flex items-center justify-center gap-2 py-2
```

Source: D-04 + existing `pollingDiagnostic` indicator pattern in `question-form.tsx` lines 298–302.

| Element | Copy |
|---------|------|
| Transition message (during 2s delay) | "Your graph is growing..." |
